# Product Guidelines

## Design Principles
1.  **Minimalist & Clean**: Maintain a focused UI with ample whitespace and clear typography to reduce cognitive load during complex tasks like receipt scanning.
2.  **Speed & Efficiency**: Optimize all user flows—especially the OCR upload and marketplace listing—for maximum speed. Minimize the number of clicks required for common actions.
3.  **Transparency & Trust**: Provide clear, real-time feedback on fraud scoring, receipt verification status, and dispute resolution to build user confidence.
4.  **Seamless Mobile Integration**: Ensure a consistent experience between the Next.js web application and the Expo mobile app, with synchronized states and intuitive cross-platform navigation.

## Visual Language
- **Color Palette**: 
  - **Green Accents**: Used for growth-related actions (monetization, successful sales, positive balances).
  - **Blue Accents**: Used for security-related elements (verification, authentication, trust badges).
- **Default Mode**: **Dark Mode Focus**—prioritize a high-end, modern dark theme suitable for tech-savvy users, while ensuring accessibility and readability.

## Tone & Voice
- **Tone**: **Professional & Direct**. Communications should be concise, clear, and helpful.
- **Voice**: Avoid jargon and conversational filler. Use active voice and direct instructions (e.g., "Scan receipt," "Confirm payout," "Review dispute").

## Interaction Standards
- **Feedback Loops**: Every action (upload, sale, dispute) must provide immediate visual confirmation.
- **Error Handling**: Error messages must be professional, explaining *what* went wrong and *how* to fix it without technical jargon.
- **Mobile-First**: Design all new features with a mobile-first mindset, ensuring touch-friendly targets and responsive layouts.
